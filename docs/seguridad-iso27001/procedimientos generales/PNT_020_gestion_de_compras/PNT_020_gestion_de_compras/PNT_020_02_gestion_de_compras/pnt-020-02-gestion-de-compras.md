# Objetivo

El objetivo del presente procedimiento es definir la operativa para
realizar las compras en Spika Tech, en adelante Spika.

# 

# Alcance

Este procedimiento se aplica a todas las adquisiciones realizadas por
SPIKA TECH que puedan tener impacto sobre:

- La calidad del producto o servicio médico

- La seguridad de la información (confidencialidad, integridad,
  disponibilidad)

- La operación de los sistemas IT utilizados en el desarrollo,
  fabricación o prestación del servicio.

#

Es responsabilidad de Garantía De Calidad supervisar el cumplimiento de

# Definiciones y Acrónimos

- IT (Information Technology): Tecnología de la Información

- Proveedor crítico: Proveedor cuyo producto o servicio puede
  afectar al cumplimiento normativo, la seguridad del paciente o la
  seguridad de la información

- ISO 13485: Norma internacional para sistemas de gestión de
  calidad en productos sanitarios

- ISO/IEC 27001: Norma internacional para sistemas de gestión de
  seguridad de la información

- NDA: Acuerdo de confidencialidad (Non-Disclosure Agreement)

- Riesgo de seguridad: Posibilidad de que un bien o servicio
  afecte negativamente a la confidencialidad, integridad o
  disponibilidad de la información

# 

# Procedimiento

# **Identificación de la necesidad**

- Descripción técnica del bien o servicio

- Justificación

- Impacto esperado (calidad, seguridad, IT)

- Requisitos funcionales y técnicos

# Evaluación de impacto y riesgos

- Se realiza un análisis de riesgos si el bien/servicio afecta:

- Al producto médico

- A la seguridad de la información

- A los sistemas críticos de IT

- La evaluación es documentada por el Responsable de Calidad o de
  Seguridad de la Información, según el caso.

# **Selección y evaluación de proveedores**

El proveedor debe estar previamente evaluado según el procedimiento de
evaluación de proveedores, si se considera crítico para el producto
desarrollado por SPIKA.

En caso de proveedor crítico, se deben considerar:

- Certificaciones ISO

- Acuerdos de confidencialidad (NDA)

- Políticas de seguridad

- Historial de cumplimiento

# Solicitud de presupuestos

El Responsable de Compras solicita mínimo dos presupuestos. Se permite
excepción justificada por disponibilidad o especialización. El
Departamento IT revisa requisitos técnicos y compatibilidad. Calidad y
Seguridad de la Información revisan si corresponde.

# Aprobación de la adquisición

La aprobación se otorga por el Responsable de Compras y/o Dirección,
tras validar:

- Requisitos técnicos

- Evaluación de riesgos

- Conformidad normativa

# Recepción y verificación

Una vez recibido el producto o servicio:

- IT realiza pruebas y verifica especificaciones

- Calidad verifica cumplimiento si aplica al sistema de calidad

- Seguridad de la Información valida medidas acordadas (contraseñas,
  accesos, etc.)

# Registro y documentación

Toda la documentación relativa al proceso de compra como pueden ser el
propio pedido,

documentos de servicios contratados, presupuestos, contratos, ofertas o
recibos, se

archivarán dentro de los sistemas de SPIKA TECH en el sharepoint
“Dirección/Finanzas y

Presupuestos/Facturas”.
