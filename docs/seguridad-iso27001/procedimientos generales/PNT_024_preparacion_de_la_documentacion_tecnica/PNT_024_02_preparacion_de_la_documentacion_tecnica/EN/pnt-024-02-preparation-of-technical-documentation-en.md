# Aim

This procedure describes the process followed by Spika Tech, hereinafter
SPIKA, for the preparation of technical documentation for medical
devices, ensuring compliance with applicable regulations in force.

# Scope

This procedure applies to the documentation generated (technical file)
on a medical device generated at SPIKA facilities.

# Responsibilities and Departments Affected

The Technical Manager will be responsible for reviewing and approving
the technical file, ensuring that all manufacturing and quality control
of the medical device has been carried out in compliance with all
applicable requirements (regulatory, internal, and customer
requirements). The technical file will be prepared by competent
personnel with knowledge of the product, its characteristics, and the
production and quality control process.

# Definitions and Acronyms

- Technical file : Set of technical documentation that allows the
 assessment of conformity with the applicable regulatory requirements
 according to current regulations.

# Procedure

# Description and generalities

For each product or family of Medical Devices designed and manufactured
by SPIKA, a Technical File must be generated, which must be periodically
reviewed and maintained with post-marketing monitoring data, so that
compliance with the applicable requirements indicated in Regulation
746/2017 and/or Regulation 745/2017, and section 4.2.3 of the ISO
13485:2016 standard is evidenced.

The Technical File will be recorded in the template included in **Annex
1** of this procedure.

# Review and approval of technical file

The Technical File will be reviewed and updated when sufficient data
from the product's monitoring or post-marketing phase is available, or
when relevant data is available, to ensure its suitability for the
product's use.
