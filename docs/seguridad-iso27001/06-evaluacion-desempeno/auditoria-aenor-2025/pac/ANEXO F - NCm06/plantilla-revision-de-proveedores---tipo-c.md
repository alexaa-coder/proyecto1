## CLAUSULAS DE PROTECCIÓN DE PROYECTOS IT – (empresa)









**REQUERIMIENTOS SEGURIDAD DE LA INFORMACIÓN**
**SI**
**NO**




El proveedor tiene acuerdos/cláusulas de confidencialidad para todo
su personal y subcontratistas afectados por los servicios contractuales
de SPIKA TECH.
 
 


El proveedor tiene un programa de formación sobre la seguridad de la
información para todo el personal.
 
 


El proveedor sigue un estándar para la administración de la
seguridad de la información y los controles de seguridad de la
información como ISO / IEC27001 / 2, CSC 20 CIS o similar, para proteger
la información en términos de confidencialidad e integridad, así como de
disponibilidad.
 
 


El proveedor tiene un proceso de gestión de cambios documentado y
estructurado para cambios en los sistemas.
 
 


El proveedor dispone de procedimientos de continuidad de negocio,
recuperación y gestión de crisis.
 
 


El proveedor tiene un proceso de gestión de incidentes documentado y
estructurado para incidentes de seguridad y violaciones de datos.
 
 


El proveedor tiene un proceso de gestión de acceso documentado y
estructurado para acceder a los sistemas.
 
 


El proveedor tiene un proceso de gestión de vulnerabilidades
documentado y estructurado que incluye análisis periódicos de
vulnerabilidades y pruebas de penetración, así como la mitigación
oportuna (de conformidad con las instrucciones de seguridad de IT de
SPIKA) de las vulnerabilidades identificadas.
 
 


Si la prestación de servicios incluye comunicaciones a través de
redes públicas, el proveedor utiliza protocolos de cifrado como TLS,
SSL, SFTP, VPN de cliente, VPN de Site2Site o tecnologías similares para
la transferencia de datos hacia y desde el proveedor.
 
 


Si la prestación de servicios incluye un sistema o aplicación
disponible desde Internet, el proveedor ha realizado un test de
penetración el proveedor puede proporcionar evidencias del mismo para su
revisión.
 
 


Si la prestación de servicios incluye un sistema o aplicación
disponible desde Internet, el proveedor ofrece una solución de inicio de
sesión único Solución Active Directory / IDaaS
