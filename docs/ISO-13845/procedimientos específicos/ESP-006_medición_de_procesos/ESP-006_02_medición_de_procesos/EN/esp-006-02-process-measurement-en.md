Establish a procedure to define, collect, analyze and use key performance indicators (KPIs)
that allow measuring the effectiveness and efficiency of software development processes at SPIKA TECH, hereinafter SPIKA.
## **2. Scope**

## **5. Procedure** **5.1. Generalities**
KPIs should be defined in accordance with the organization's quality objectives and should be :
- Specific : Clear and well defined.
- Measurable : Quantifiable to allow monitoring.
- Achievable : Realistic and possible to achieve.
- Relevant : Directly related to quality objectives.
- Temporary : Defined in a specific time frame.
## **5.2. Creating a KPI**
For the correct definition and implementation of a KPI, it is recommended to follow these steps:
- Process Identification : Determine the process you want to measure.
- Definition of the Objective : Establish what you want to achieve with the measurement.
- Indicator Selection : Choose a KPI that adequately reflects process performance.
- Establishing Measurement Methods : Determine how data will be collected and
analyzed.
- Measurement Frequency : Define how regularly the KPI will be evaluated.
The definition of KPIs should be consistent with the term SMAT, as indicated here:

## **5.3. Indicator registration**
KPIs should be documented in a log that includes:
- KPI Name
- Description
- Calculation Formula
- Reference Value
- Measurement Frequency
- Responsible
## **5.4. Data analysis**
The data collected must be analyzed to identify trends, deviations, and opportunities for improvement. This analysis must be documented and used to make informed decisions
about process improvements.
