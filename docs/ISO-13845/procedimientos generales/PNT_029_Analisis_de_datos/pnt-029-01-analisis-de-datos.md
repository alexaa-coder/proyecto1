# Objetivo

Establecer el proceso para determinar, recopilar, analizar e interpretar
los datos necesarios para demostrar la idoneidad, adecuación y eficacia
del sistema de gestión de la calidad (SGC) de Spika Tech, en adelante
SPIKA, así como para identificar oportunidades de mejora basadas en
datos objetivos y fiables.

# Alcance

Este procedimiento aplica a todas las áreas de SPIKA que generan,
utilizan o analizan datos relacionados con el desempeño del SGC,
incluyendo la satisfacción del cliente, la conformidad de los productos,
el desempeño de los procesos, proveedores, auditorías, reclamaciones y
cualquier otra fuente relevante.

# Definiciones y Acrónimos

-   **SGC**: Sistema de Gestión de Calidad de SPIKA.

# Procedimiento

El presente procedimiento describe de forma estructurada las etapas
necesarias para garantizar un análisis eficaz de los datos que alimentan
el sistema de gestión de la calidad (SGC) de Spika. Su objetivo es
proporcionar una base objetiva para la toma de decisiones, la mejora
continua y la conformidad con los requisitos establecidos, tanto
regulatorios como internos.

# Determinación de datos a analizar

Se identificarán fuentes internas y externas de datos que permitan
evaluar el desempeño del SGC, incluyendo:

-   Retroalimentación del cliente (PNT-009).

-   Conformidad de los productos (resultados de inspecciones, ensayos y
    validaciones).

-   Características y tendencias de los procesos (KPIs definidos).

-   Resultados de auditorías internas y externas (PNT-013).

-   Indicadores de proveedores (plazos, calidad, incidencias).

-   Reclamaciones de clientes (PNT-007).

-   No conformidades y acciones correctivas/preventivas (PNT-004 y
    PNT-006).

-   Informes técnicos de asistencia o soporte postventa, cuando
    apliquen.

# Recopilación de datos

Cada proceso debe documentar los métodos de recopilación de datos. Los
datos deben ser:

-   Representativos y trazables.

-   Recopilados en tiempo y forma definidos.

-   Almacenados en soportes adecuados (bases de datos, hojas de cálculo,
    formularios, ect).

# Análisis de datos

El análisis de datos se realizará con las salidas del seguimiento y
medición, así como cualquier fuente relevante y debe incluir como mínimo
la siguiente información:  

-   Retroalimentación 

-   Conformidad de los requisitos del producto 

-   Característica y tendencia de los procesos, incluyendo oportunidades
    de mejora 

-   Proveedores 

-   Auditorias 

-   Informes de asistencia técnica, cuando aplique 

Como resultado de este análisis si se originan no idoneidades en el
sistema, este análisis será el punto de partida en la oportunidad de
mejora.
